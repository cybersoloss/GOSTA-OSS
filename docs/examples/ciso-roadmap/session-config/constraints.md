# Session Constraints: CISO Priorities 2026

## Hard Constraints
1. **No vendor-specific recommendations.** The assessment must be product-agnostic. Mentioning capability categories (EDR, MFA, SIEM) is permitted; naming specific vendors is not.
2. **EU-specific grounding.** All regulatory, threat, and operational claims must be grounded in EU context. US-centric data may be cited only when EU-specific data is unavailable, with explicit annotation.
3. **Mid-market feasibility.** Every priority must be executable by an organization with 50-500 employees, 0.5-3 security FTE, and €50K-€300K annual security budget.
4. **No implementation guides.** The deliverable is a priority assessment with reasoning, not a how-to document. Implementation details belong to a separate scope.

## Soft Constraints
1. **Temporal relevance.** Priorities should reflect the 2026 regulatory and threat landscape, not historical conditions. Regulatory status (NIS2 transposition) should reflect current state.
2. **Actionable sequencing.** The priority list should be sequenced for execution in annual batches (3-4 items/year), not presented as a simultaneous checklist.

## Publication Constraints
1. **Gartner copyright.** The Gartner reference pool (328 items) is used during deliberation for internal evidence grounding only. No Gartner report titles, excerpts, IDs, or paraphrased findings attributed to Gartner shall appear in any public artifact. In deliberation transcripts prepared for publication, Gartner-sourced claims must be generalized (e.g., "analyst research indicates..." or "industry analysis suggests...") or replaced with publicly available equivalent sources.
2. **Blog content — fully publishable.** All 219 blog pool articles are the publishing organization's own curated content and may be reproduced, excerpted, and cited freely in public artifacts.
3. **DBIR public citation.** Verizon DBIR 2025 is publicly available. Statistics may be cited with attribution ("Verizon DBIR 2025"). Large excerpt reproduction is not permitted.
4. **Pool mechanism transparency.** Public artifacts may describe the reference pool mechanism (query terms, relevance scores, item counts, pool sizes) without reproducing copyrighted pool content. This demonstrates GOSTA's retrieval capability without IP exposure.

## Scope Exclusions
- Sector-specific requirements (healthcare, energy, financial services beyond generic NIS2)
- Organizational transformation or culture change programs
- Detailed technical architecture design
- Specific budget allocation recommendations (beyond order-of-magnitude)
- OT/ICS security (significant enough to be a separate scope)
